What's going on? (Photoshop analogy)

Think of the page as a Photoshop file with layers. Certain CSS properties (position: sticky, backdrop-filter) force the browser to put an element on its own layer. A child element can only blur things on its own layer — not layers underneath.

  • Layer 1: Page content (bible text, images)
  • Layer 2: .menu — gets its own layer because of sticky + backdrop-filter

Broken: #controlPanel is a child of .menu, so it's stuck on Layer 2. Its blur works where it overlaps the nav bar (also Layer 2), but below that there's nothing on Layer 2 to blur. The bible text is on Layer 1 — unreachable.

Fixed: #controlPanel is moved outside .menu as a sibling. It gets its own layer which can blur Layer 1 directly. Both are wrapped in .menu-wrapper which handles sticky positioning without creating a compositing conflict.
